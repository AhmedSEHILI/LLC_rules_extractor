Submodules¶

views.AccueilPage module¶

class views.AccueilPage.AccueilPage(parent=None)¶

Bases : QWidget

Page d’accueil de l’application.

Cette vue affiche un message de bienvenue centré, ainsi qu’un bouton permettant d’ouvrir une page Wikipédia sur la Leucémie lymphoïde chronique (LLC).

ouvrir_lien_llc()¶

Ouvre une page Wikipédia expliquant la maladie LLC.

views.Amie3ParamsWidget module¶

class views.Amie3ParamsWidget.Amie3ParamsWidget(parent=None)¶

Bases : QWidget

Widget graphique permettant de configurer les paramètres d’exécution de AMIE3.

Il regroupe des champs de saisie textuelle pour les options numériques et des cases à cocher pour les options, et construit la liste des arguments à fournir à AMIE3.

build_amie3_params()¶

Construit et retourne la liste des paramètres AMIE3 à partir des champs de saisie et des options sélectionnées par l’utilisateur.

Renvoie:

Une liste de chaînes représentant les arguments en ligne de commande à passer à AMIE3.

Type renvoyé:

list

class views.AnalyseDonneesPage.AnalyseDonneesPage(parent=None)¶

Bases : QWidget

Vue dédiée à l’analyse et l’interrogation des données.

Cette page contient un tableau pour afficher les résultats des requêtes sur l’ontologie médicale.

views.ComparaisonPage module¶

class views.ComparaisonPage.ComparaisonPage(parent=None)¶

Bases : QWidget

Page d’interface pour comparer des ensembles de règles.

Cette vue affiche deux panneaux côte à côte : - à gauche, les règles chargées depuis un fichier ; - à droite, les règles générées automatiquement par AMIE3.

views.ExtractionReglesPage module¶

class views.ExtractionReglesPage.ExtractionReglesPage(parent=None)¶

Bases : QWidget

Page dédiée à l’extraction, l’affichage et la gestion des règles.

Cette vue permet de présenter les résultats de l’extraction de règles (issues d’AMIE3) dans un tableau.

views.FenetreGraphe module¶

class views.FenetreGraphe.FenetreGraphe(image_path=None, titre="Graphe de l'ontologie")¶

Bases : QWidget

Fenêtre d’affichage graphique d’une ontologie.

Cette classe affiche une image (un graphe RDF/OWL) dans une fenêtre avec des contrôles de zoom avant, zoom arrière et réinitialisation.

afficher_graphe(image_path)¶

Définit une nouvelle image à afficher et ouvre la fenêtre.

Paramètres:

image_path (str) – Chemin de l’image à afficher.

afficher_image()¶

Charge et affiche l’image à partir du chemin image_path.

reset_zoom()¶

Réinitialise le zoom de l’image à son échelle d’origine.

update_pixmap()¶

Met à jour l’affichage de l’image avec le facteur de zoom actuel.

zoom_in()¶

Effectue un zoom avant sur l’image.

zoom_out()¶

Effectue un zoom arrière sur l’image.

class views.GestionOntologiesPage.GestionOntologiesPage(parent=None)¶

Page de l’application dédiée à la gestion des ontologies.

views.QualiteValidationPage module¶

class views.QualiteValidationPage.QualiteValidationPage(parent=None)¶

Bases : QWidget

Page dédiée à la qualité et à la validation des règles extraites.

Cette vue permet d’afficher les règles dans un tableau afin de mesurer leur qualité (selon des métriques spécifiques) et de les valider ou non.

views.RequeteSPARQLView module¶

class views.RequeteSPARQLView.RequeteSPARQLView(mere=None)¶

views.RuleExtractionView module¶

class views.RuleExtractionView.RuleExtractionView¶

Bases : QMainWindow

Fenêtre principale de l’application d’extraction de règles à partir d’ontologies.

Cette vue intègre toutes les pages fonctionnelles de l’application (chargement d’ontologies, extraction, validation, comparaison…) via un QStackedWidget et une interface graphique organisée autour d’un panneau latéral de navigation.

afficher_graphe(image_path)¶

Affiche une image de graphe dans une fenêtre dédiée.

Paramètres:

image_path (str) – Chemin de l’image à afficher.

views.RuleSetTestView module¶

class views.RuleSetTestView.RuleSetTestView(mere=None)¶

Bases : QMainWindow

Fenêtre de saisie et de mesure de qualité pour un ensemble de règles.

Permet à l’utilisateur d’entrer plusieurs règles (une par ligne), puis de déclencher leur évaluation à l’aide d’un bouton.

nettoyer()¶

views.RuleTestview module¶

class views.RuleTestview.RuleTestView(mere=None)¶

Bases : QMainWindow

Fenêtre de saisie d’une règle individuelle.

Cette interface permet à l’utilisateur d’entrer une seule règle et de consulter les classes et propriétés disponibles dans l’ontologie.
